      Kubernetes Autoscaling

      1️⃣ 오토스케일링(AutoScaling) 개요

      쿠버네티스에서는 부하에 따라 리소스를 자동으로 확장(AutoScaling) 가능.
      ✅ 트래픽 증가 시 자동 확장 → 성능 유지
      ✅ 트래픽 감소 시 자동 축소 → 비용 절감

      쿠버네티스에서 제공하는 오토스케일링 방식:

      1. HPA (Horizontal Pod Autoscaler) → Pod 개수 자동 확장
      2. VPA (Vertical Pod Autoscaler) → Pod의 CPU & 메모리 할당량 자동 조정
      3. Cluster Autoscaler → 노드(Node) 수 자동 확장

      🔽 쿠버네티스 오토스케일링 개요

      2️⃣ HPA (Horizontal Pod Autoscaler)

      HPA는 CPU, 메모리 사용량을 기반으로 Pod 개수를 자동으로 조정.

      1. HPA 동작 방식

      ✅ Metrics Server가 Pod 리소스 사용량 수집
      ✅ HPA 컨트롤러가 리소스 사용량 확인
      ✅ 트래픽 증가 시 Pod 추가, 감소 시 Pod 제거

      🔽 HPA 아키텍처 흐름도

      2. HPA 설정 예제

      ✅ Deployment (my-app) 배포

      apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
        - name: my-app
          image: nginx
          resources:
            requests:
              cpu: "100m"
            limits:
              cpu: "200m"

      ✅ HPA 생성 (CPU 50% 이상 사용 시 확장)

      apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: my-app-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 1
  maxReplicas: 10
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 50

      ✅ HPA 적용 후 확인

      kubectl apply -f hpa.yaml
kubectl get hpa

      3️⃣ VPA (Vertical Pod Autoscaler)

      VPA는 Pod 자체의 CPU & 메모리 할당량을 자동으로 조정.

      1. VPA 동작 방식

      ✅ VPA 컨트롤러가 Pod 리소스 사용량 분석
      ✅ 적절한 CPU & 메모리 할당량을 추천
      ✅ Pod 재시작 후 새로운 리소스 설정 적용

      🔽 VPA 아키텍처 흐름도

      2. VPA 설정 예제

      ✅ VPA 설치 (Metrics Server 필요)

      kubectl apply -f https://github.com/kubernetes/autoscaler/releases/latest/download/vertical-pod-autoscaler.yaml

      ✅ VPA 설정 (my-app에 적용)

      apiVersion: autoscaling.k8s.io/v1
kind: VerticalPodAutoscaler
metadata:
  name: my-app-vpa
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  updatePolicy:
    updateMode: Auto

      ✅ VPA 적용 후 리소스 추천 확인

      kubectl describe vpa my-app-vpa

      4️⃣ Cluster Autoscaler

      Cluster Autoscaler는 노드 수를 자동으로 조정하는 기능.
      ✅ HPA가 Pod 개수를 늘려도 노드가 부족하면? → Cluster Autoscaler가 새로운 노드 추가
      ✅ Pod이 줄어들어 노드가 불필요하면? → 자동으로 노드 제거

      🔽 Cluster Autoscaler 아키텍처

      1. Cluster Autoscaler 설정

      ✅ AWS EKS에서 Autoscaler 설치

      helm repo add autoscaler https://kubernetes.github.io/autoscaler
helm install cluster-autoscaler autoscaler/cluster-autoscaler

      ✅ Cluster Autoscaler 설정 파일

      apiVersion: apps/v1
kind: Deployment
metadata:
  name: cluster-autoscaler
spec:
  template:
    spec:
      containers:
        - name: cluster-autoscaler
          image: k8s.gcr.io/autoscaling/cluster-autoscaler:v1.21.0
          command:
            - ./cluster-autoscaler
            - --cloud-provider=aws
            - --nodes=1:10:my-node-group

      ✅ 클러스터 노드 상태 확인

      kubectl get nodes
kubectl describe nodes

      ✅ 결론

      쿠버네티스 오토스케일링을 활용하면 부하에 맞춰 리소스를 자동으로 조절 가능.
      1️⃣ HPA → Pod 개수 자동 확장
      2️⃣ VPA → Pod 리소스 자동 조정
      3️⃣ Cluster Autoscaler → 노드 자동 확장
